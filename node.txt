ROS2 rqt_graph issue in WSL

Problem:
Running `rqt_graph` shows error:
QStandardPaths: error creating runtime directory '/run/user/1000/' (Permission denied)

Reason:
`rqt_graph` is a Qt GUI tool. Qt requires a runtime directory (`XDG_RUNTIME_DIR`).
In WSL this directory is not automatically created, so the GUI cannot start.

Fix:

sudo mkdir -p /run/user/1000
sudo chown pradeep:pradeep /run/user/1000
chmod 700 /run/user/1000
export XDG_RUNTIME_DIR=/run/user/1000

Purpose:
Creates the runtime directory and gives permission so Qt-based ROS tools like `rqt_graph` can run.

Usage:

1. ros2 run demo_nodes_cpp talker
2. ros2 run demo_nodes_cpp listener
3. rqt_graph
